## 一、功能概述

### 1.1 一句话描述

在 yiyan.baidu.com 工具页新增「剧本诊断」功能，用户上传/粘贴短剧剧本文本，系统自动生成一份结构化的**短剧商业潜力评估报告**，涵盖 4 大板块 17 个维度的评分、分析与改进建议。

### 1.2 目标

| 目标 | 衡量指标 |
|------|---------|
| 创作者能快速获得专业级剧本诊断 | 从上传到报告产出 ≤ 90s |
| 报告有实际参考价值 | 用户满意度 ≥ 3.5/5（报告底部评价） |
| 工具有复访价值 | 7日复访率 ≥ 15% |

### 1.3 用户

编剧、短剧创作者、网文作者（想改编为短剧的）。

## 二、功能结构

```
剧本诊断（工具页）
├── 输入区
│   ├── 文本粘贴输入
│   ├── 文件上传（.txt / .docx）
│   └── 示例剧本（一键体验）
├── 诊断中（进度状态）
├── 诊断报告
│   ├── 总分卡片

│   ├── 雷达图（17维度）

│   ├── 详细分析（可展开/收起）
│   │   ├── A. 叙事与剧本基因（7维度）
│   │   ├── B. 市场共鸣与竞争定位（6维度）
│   │   ├── C. 合规性评估（2维度）
│   │   └── D. 商业化与ROI展望（2维度）

| G-4 | 满意度评价 | 4 个表情选项，单选。选择后替换为"感谢反馈！"并上报埋点。仅可评价一次。 |

### 4.2 接口定义

#### 接口1：创建诊断任务

```
POST /api/v1/script-eval/create

Request:
{

  "code": 0,

    "eval_id": "abc123",
    "status": "step_3",

      { "name": "识别剧本信息", "status": "completed" },

      { "name": "生成改稿建议", "status": "pending" }

  }

  "code": 0,
  "data": {

轮询策略：
- 前端每 3 秒轮询一次

## 五、后端处理流程

```
                 POST /create
                      │
                      ▼
           创建任务（status: pending）
                      │
                      ▼
              推入异步任务队列
                      │
    ┌─────────────────┴─────────────────┐
    │           Worker 执行               │
    │                                    │
    │  Step 1: 提取基础信息               │
    │    └→ 更新 status: step_1           │
    │    └→ LLM 调用 (Prompt 1)          │
    │                                    │
    │  Step 2-5: 四大板块评估（并发）       │
    │    └→ 更新 status: step_2           │
    │    └→ 4 个 LLM 调用并发执行          │
    │      ├─ A.叙事 (Prompt 2)           │
    │      ├─ B.市场 (Prompt 3)           │
    │      ├─ C.合规 (Prompt 4)           │
    │      └─ D.商业 (Prompt 5)           │
    │    └→ 每完成一个更新 progress        │
    │                                    │
    │  Step 6: 汇总 + 生成建议            │
    │    └→ 加权计算总分                  │
    │    └→ 一票否决检查                  │
    │    └→ LLM 调用 (Prompt 6)          │
    │    └→ 更新 status: completed        │
    │                                    │
    └────────────────────────────────────┘
```

**LLM 调用规格**：

| 步骤 | 调用数 | 并发 | 预估耗时 | 备注 |
|------|--------|------|---------|------|
| Step 1 基础信息 | 1 | - | 5-10s | 仅发送前 3 万字 |
| Step 2-5 四板块 | 4 | 4路并发 | 15-30s | 每路约 8 万字输入 |
| Step 6 建议 | 1 | - | 5-10s | 输入为各板块得分摘要 |
| **合计** | **6** | | **25-50s** | |

**容错机制**：

| 场景 | 处理 |
|------|------|
| 单个 LLM 调用超时（>60s） | 自动重试 1 次 |
| LLM 返回内容 JSON 解析失败 | 自动重试 1 次，仍失败则该维度返回默认分值并标注"解析异常" |
| 某个板块评估整体失败 | 该板块返回默认中位分值（70），总分仍可计算，报告标注"部分维度评估异常" |
| 全部失败 | 返回失败态，引导用户重试 |

**隐私保护**：

| 规则 | 说明 |
|------|------|
| 剧本原文不落库 | 仅在任务处理的内存链路中传递，处理完成后不持久化存储 |
| 结果 JSON 落库 | 评估结果（不含原文）存数据库，用于历史记录和数据分析 |
| 用户可删除 | 用户可删除自己的诊断记录 |

## 七、边界情况与特殊处理

| 场景 | 处理方式 |
|------|---------|
| 输入内容不是剧本（如小说、新闻、代码等） | 不做前置拦截。LLM 评估时会自然反映在低分中，核心结论中会指出"该文本不符合短剧剧本格式" |
| 输入纯英文剧本 | 正常评估，LLM 支持中英文 |
| 输入内容含大量特殊字符/乱码 | 前端做基础清洗（去除连续空行、控制字符）。若清洗后 < 500 字，提示内容不足 |
| .docx 文件解析失败 | toast 提示"文件解析失败，请尝试复制文本内容直接粘贴" |
| 用户未登录 | 可以进入页面和输入内容，点击「开始诊断」时弹出登录框 |
| 用户连续快速提交 | 同一用户同时只能有 1 个进行中的诊断任务。重复提交返回错误码，前端提示"有诊断正在进行中" |
| 剧本内容有严重违规（黄暴等） | LLM 自身的安全机制会拦截，返回无法评估。前端展示"该内容无法评估，请检查剧本内容" |
