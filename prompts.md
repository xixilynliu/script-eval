# 剧本评估 Prompt 模板

以下 prompt 按评估步骤组织，由 `script_eval.py` 自动调用。每个 prompt 设计为独立调用，避免单次 token 过长。

---

## Prompt 1：基础信息提取

```
你是一个专业的剧本分析师。请阅读以下剧本文本，提取基础信息。

【剧本文本】：
{script_text}

请严格按以下 JSON 格式输出，不要输出任何其他内容：

{{
  "title": "剧本名称（从文本中识别，如无法识别则填'未知'）",
  "genre": "频类（女频/男频/通用）",
  "themes": ["题材标签1", "题材标签2", "..."],
  "episode_count": "可识别的集数（整数，无法识别填-1）",
  "main_characters": [
    {{
      "name": "角色名",

      "description": "一句话角色定位"

  ],

    "night": ["夜景场景1", "夜景场景2"]

## Prompt 2：叙事与剧本基因评估（A板块）

```
你是一个资深的短剧商业评估专家。请对以下剧本进行「叙事与剧本基因」维度的深度评估。

【剧本基础信息】：
{basic_info_json}

【剧本文本】：
{script_text}

请从以下 7 个维度逐一评估，每个维度必须包含：评分（0-100）、3条分析要点、至少2条可打磨建议。

评分标准参考：
- 90-100 (S级)：该维度表现卓越
- 80-89 (A级)：该维度表现优秀
- 70-79 (B级)：该维度表现中等
- 60-69 (C级)：该维度表现较弱
- <60 (D级)：该维度表现差

**维度1：钩子强度（权重20%）**
评估重点：前30秒/第一集开篇吸引力、悬念设置强度、角色魅力展现速度、集尾钩子驱动力。
短剧特别要求：开篇必须在极短时间内建立核心冲突，不允许慢热。

**维度2：爽点密度（权重20%）**
评估重点：爽点总数与集均密度、爽点类型覆盖（情感/视觉/反转/打脸/技能展示）、前5集分布密度、升级递进感。
短剧特别要求：每集至少1个明确爽点，前3集密度应高于平均。

**维度3：节奏与步调（权重15%）**
评估重点：短剧节奏适配度、单场戏时长控制、对话密度和信息释放节奏、场景转换流畅度。
短剧特别要求：拒绝长剧式慢节奏铺垫，信息要高效释放。

**维度4：情节连贯性（权重15%）**
评估重点：主线因果链清晰度、人物动机一致性、伏笔呼应处理、时间线处理。

**维度5：角色吸引力（权重15%）**
评估重点：主角共鸣点和记忆度、男/女主角魅力鲜明度、配角功能性与立体感、人物关系网络复杂度。

**维度6：对白质量（权重8%）**
评估重点：风格与题材匹配度、金句密度和网感/传播性、人物语言区分度、台词长度适配短剧节奏。

**维度7：悬念有效性（权重7%）**
评估重点：集尾钩子设计、多层次悬念布局、悬念-解答循环节奏、视觉化悬念。

请严格按以下 JSON 格式输出：

{{
  "hook_strength": {{
    "score": 72,
    "grade": "B",
    "analysis": ["分析要点1", "分析要点2", "分析要点3"],
    "improvements": ["可打磨建议1", "可打磨建议2"]
  }},
  "excitement_density": {{
    "score": 75,
    "grade": "B",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }},
  "pacing": {{
    "score": 74,
    "grade": "B",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }},
  "plot_coherence": {{
    "score": 82,
    "grade": "A",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }},
  "character_appeal": {{
    "score": 80,
    "grade": "A",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }},
  "dialogue_quality": {{
    "score": 76,
    "grade": "B",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }},
  "suspense_effectiveness": {{
    "score": 77,
    "grade": "B",
    "analysis": ["...", "...", "..."],
    "improvements": ["...", "..."]
  }}
}}

要求：
1. 每条分析要点必须有具体的剧本内容引用或详细说明，不可泛泛而谈
2. 可打磨建议必须具体可执行，指出具体修改方向
3. 评分要求客观严格，不可偏高；短剧节奏不达标的需严格扣分
4. 仅输出 JSON，不要输出其他内容
```

**维度1：爆款契合度（权重25%）**
评估重点：题材是否处于市场热度上升期、核心情感元素是否齐全（对标近期爆款短剧）、是否有成功的对标作品。

- 女频热门：穿越重生、打脸虐渣、逆袭、霸总甜宠、虐恋情深、宫斗权谋

## Prompt 4：合规性评估（C板块）

```
你是一个专业的内容合规审核专家。请对以下剧本进行合规性评估。

【剧本基础信息】：
{basic_info_json}

【剧本文本】：
{script_text}

请从以下 2 个维度评估：

**维度1：内容合规性（权重60%）**
检查项：
1. 政治安全：是否涉及敏感历史事件/政治影射/民族宗教问题
2. 暴力尺度：暴力描写是否过度血腥或美化暴力
3. 性内容尺度：是否存在过度暴露/直接色情内容
4. 未成年人保护：涉及未成年角色的内容是否得当
5. 社会影响：是否宣扬不良价值观/违法行为

**维度2：价值观导向（权重40%）**
检查项：
1. 情感价值观是否正向
2. 家庭伦理观念是否健康
3. 权力/金钱观念是否理性
4. 性别形象塑造是否积极
5. 主角行为是否具有正面示范效应

请严格按以下 JSON 格式输出：

{{
  "content_compliance": {{
    "score": 88,
    "grade": "A+",
    "analysis": ["...", "...", "..."],
    "risk_items": ["需关注的风险项1（如有）"],
    "improvements": []
  }},
  "value_orientation": {{
    "score": 85,
    "grade": "A+",
    "analysis": ["...", "...", "..."],
    "improvements": []
  }}
}}

要求：
1. 合规性评估必须严格客观，宁严勿松
2. 明确指出每一个潜在风险点及其所在位置
3. 无风险的维度也要说明为什么安全
4. 仅输出JSON
```

## Prompt 5：商业化与ROI评估（D板块）

```
你是一个资深的短剧商业分析专家。请对以下剧本进行商业化潜力评估。

【剧本基础信息】：
{basic_info_json}

【剧本文本】：
{script_text}

请从以下 2 个维度评估：

**维度1：制作难度（权重40%）**
评估维度：
1. 场景数量和复杂度（分日景/夜景列出）
2. 演员要求（主演数量、演技要求、是否需要知名演员）
3. 特效/动作需求
4. 服化道投入（现代低 < 古装高 < 奇幻最高）
5. 群演需求
6. 预估制作周期和难度等级（低/中/高/极高）

**维度2：变现潜力（权重60%）**
评估维度：
1. 付费点设计空间：哪些情节节点适合设为付费解锁点（具体到集数）
2. 广告植入适配度
3. 打赏/互动变现潜力
4. 可商业化元素（特色道具/服装/场景等）

请严格按以下 JSON 格式输出：

    "analysis": ["...", "...", "..."],

    }},

  }},
  "monetization_potential": {{

    "paywall_suggestions": ["第X集末尾：具体情节描述", "..."],

}}

要求：
1. 制作难度评估要具体到场景和需求明细
2. 付费点建议要结合剧情的高潮/悬念节点，指出具体集数
3. 评分要考虑短剧的成本敏感性，制作难度高对应低分（难度越高分越低）
4. 仅输出JSON

## Prompt 6：执行摘要与建议生成

```
你是一个资深的短剧商业评估专家。请基于以下各维度的评估结果，生成执行摘要和可操作建议。

【剧本基础信息】：
{basic_info_json}

【各板块评估结果】：
{all_scores_json}

【总体评分】：{total_score}/100（{total_grade}级）

请生成：

1. **核心结论**（200字以内）：总结该剧本的核心优势和主要问题，给出是否值得投入的判断。

2. **可操作建议**（5-8条）：基于各维度的"可打磨点"，归纳最重要的改进建议。要求：
   - 按优先级排序（最影响商业价值的排前面）
   - 每条建议必须具体可执行
   - 包含改进方向和预期效果

请严格按以下 JSON 格式输出：

{{
  "core_conclusion": "核心结论文本",
  "suggestions": [
    {{
      "title": "建议标题（10字以内）",
      "detail": "详细建议内容"
    }}
  ]
}}

仅输出JSON。
```

## 使用注意事项

1. 所有 prompt 中的 `{变量}` 在代码中动态替换
2. LLM 输出的 JSON 需做容错解析（可能包含 markdown 代码块标记）
3. 各板块评估 prompt 可并行调用以提升速度
4. 如剧本文本超过模型 context 限制，需做分段摘要后再评估
5. Prompt 中的示例分数仅为格式参考，不应影响 LLM 的实际评分
